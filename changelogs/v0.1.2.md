# QClaw v0.1.1 → v0.1.2 代码变化分析

> **说明**：以下分析排除了我们在逆向过程中手动打的补丁（invite code bypass、injectEnvUrls 条件修改、PROTECTED_CONFIG_PATHS 增加 wsUrl、cr() 函数禁用），仅列出 v0.1.2 官方版本相对于 v0.1.1 的**原始逻辑变化**。

---

## 一、主进程 (`out/main/index.js`)

主进程变化极小（10254 行 → 10266 行，仅多 12 行），只有 **1 处新增逻辑**：

### 1. 新增 macOS 日志目录清理

v0.1.2 在 `app.whenReady()` 回调中新增了启动时清理日志目录的逻辑：
- 新增 `rmSync` 到 fs 导入
- 在 macOS 上递归删除 `~/Library/Logs/QClaw/main/` 和 `~/Library/Logs/QClaw/openclaw/` 日志子目录
- **目的**：清理 content-security 插件在 v0.1.1 中写入的审计日志文件，防止残留

## 三、资源文件 (`resources/openclaw/config/`)

### 1. 配置模板：Gateway Auth Token 清空

**文件**：`resources/openclaw/config/openclaw.json`

```diff
 "auth": {
   "mode": "token",
-  "token": "ad264d2126b9ccfcd517f57e316d57beb73e39c24295e54f"
+  "token": ""
 }
```

- v0.1.1：所有安装实例共享同一个硬编码 gateway token
- v0.1.2：token 为空，运行时由 `ensureConfig()` 调用 `randomBytes(24).toString("hex")` 动态生成唯一 token
- **目的**：每个实例拥有独立 token，避免安全隐患

### 2. content-security 插件：日志系统全面禁用 ⭐

**这是 v0.1.2 最大的变化**，涉及 6 个源文件，共约 480 行差异。

#### `src/client.ts`（33 行差异）
- 移除 `fs`、`path` 导入（不再需要文件 I/O）
- 移除 `logger` 和 `logFile` 成员变量
- `writeLog()` 从双写实现（console + 文件追加）替换为空函数：`// 日志已禁用`

#### `src/types.ts`（15 行差异）
- 从 `CreateTaskClientOptions` 接口中移除 `logger` 和 `logFile` 配置项

#### `src/interceptor.ts`（348 行差异，最大的文件）
- 所有 `api.logger.info/warn` 和 `recordLogEvent()` 调用被注释掉
- 涉及：请求拦截摘要、历史消息清洗结果、会话释放、各阶段审计记录

#### `src/security.ts`（56 行差异）
- 禁用：降级状态探测日志、探测成功/失败日志、内容安全检查命中结果（标签/级别/类型/TraceID）、错误和降级模式日志

#### `src/utils.ts`（25 行差异）
- `recordLogEvent()` 替换为空实现（参数加 `_` 前缀，函数体为 `// 日志已禁用`）

#### `index.ts`（28 行差异）
- 移除：状态目录路径输出、配置缺失错误详情、日志文件路径构建逻辑（`content-security-YYYY-MM-DD.log`）、所有 Hook 注册成功确认日志（`✅`）、插件初始化完成日志、会话历史提取失败错误日志

### 3. 调试文件清理

移除 `src/1.txt` — 包含实际审核请求/响应内容的调试日志（含敏感内容检测示例数据）。

### 4. 无变化的部分

- **wechat-access 插件**：所有文件完全相同（index.ts、websocket/*.ts、common/*.ts、http/*.ts、websocket.md）
- **Skills 目录**：8 个技能完全相同
- **channel.json**：`{ "channel": 5001 }`，无变化

## 四、总结

v0.1.2 是一个**安全加固 + 隐私保护**版本，核心主题：

> **移除 content-security 插件的所有运行时日志（防止审计数据泄露），清理已有日志残留，消除共享 gateway token。**

| 类别 | 变化 | 影响 |
|------|------|------|
| 🔒 安全加固 | Gateway auth token 从硬编码改为运行时随机生成 | 每个实例有独立 token |
| 🔕 日志禁用 | content-security 插件 6 个文件全面禁用日志（~480 行） | 不再写审计记录到本地 |
| 🧹 日志清理 | 启动时删除 macOS 日志目录 (`main/`、`openclaw/`) | 清除 v0.1.1 残留的日志文件 |
| 🧹 文件清理 | 移除 `1.txt` 调试日志文件 | 不再包含敏感检测示例数据 |
| 📁 代码重构 | Chat 组件文件重命名 + 新增 chat-logo 图片 | 无功能影响 |
| ❌ 无变化 | wechat-access、skills、API 端点、平台服务层、auth 模块 | 功能完全一致 |

**没有新增功能、没有新增 API、没有 wechat-access 协议变化。** v0.1.2 纯粹是在 v0.1.1 基础上做"安全收尾"。
