## 第一部分：v0.1.2 → v0.1.3 代码变化分析

> **说明**：以下分析对比的是 v0.1.2 和 v0.1.3 的**原始代码**（排除我们在 v0.1.2 上打的补丁）。

---

### 一、主进程 (`out/main/index.js`)

**零逻辑变化。** v0.1.3 主进程与原始 v0.1.2 完全相同（10264 行）。

- `injectEnvUrls()` — 相同（仍使用 `!== envUrls.xxx` 条件）
- `PROTECTED_CONFIG_PATHS` — 相同（不含 `wsUrl`）
- `mergeTemplateWithProtection()` — 相同
- macOS 日志清理 — 相同
- 所有 IPC handler — 相同

### 二、渲染进程 (`out/renderer/assets/`)

**零变化。** 所有 26 个文件与原始 v0.1.2 完全相同（文件名、大小一致）。

- `index-BRreSJaD.js` — 相同（含 `cr()` 函数、invite code 校验、Chat 组件）
- `platform-QEsQ5tXh.js` — 相同（API 服务层）
- `index-DNfvYBnh.js` — 相同（auth 模块）
- 所有 CSS、图片资源 — 相同

### 三、资源文件 (`resources/openclaw/config/`)

#### 1. 配置模板 `openclaw.json` — 无变化

#### 2. wechat-access 插件 — 无变化（所有文件 md5 相同）

#### 3. Skills 目录 — 无变化（8 个技能完全一致）

#### 4. content-security 插件 — **唯一变化** ⭐

##### 变化概述

| 文件 | 变化类型 |
|------|----------|
| `index.ts` | 插件名称/描述简化，拦截提示语变更 |
| `src/security.ts` | 移除 `getBlockLevel` 导出，移除未用 import，增加 optional chaining |
| `src/interceptor.ts` | 移除标签解析逻辑，拦截消息从动态生成改为静态字符串 |
| `dist/index.js` | **新增** 编译产物（13.6KB） |

##### 详细变化

**`index.ts` — 元数据简化 + 拦截提示语变更**
- 插件名称：`"内容安全审核"` → `"内容安全"`
- 描述：`"内容安全审核插件，保护 LLM 和 Agent 生命周期免受有害内容侵害。"` → `"内容安全"`
- `before_tool_call` 拦截消息：`"内容已被安全审核拦截。"` → `"请换个问题提问。"`
- `after_tool_call` 拦截消息：同上

**`src/security.ts` — 代码精简**
- 移除 `getResultTypeName`、`getResultLevelName` 导入（不再使用）
- 移除 `getBlockLevel()` 导出函数
- `data.ResultType` → `data?.ResultType`（增加 optional chaining 防御）

**`src/interceptor.ts` — 拦截逻辑简化**
- 移除 `normalizeMessage`、`generateSecurityMessage`、`parseFirstLabels` 等工具函数引用
- 移除命中标签解析逻辑（`parseFirstLabels` + `generateSecurityMessage`）
- 拦截消息从动态生成改为静态字符串：`"<!--CONTENT_SECURITY_BLOCK-->抱歉该任务处理异常，请更换任务再尝试，为保障使用，该问答将在3秒后被删除。"`

**`dist/index.js` — 新增编译产物**
- v0.1.2 无 `dist/` 目录
- v0.1.3 新增 `dist/index.js`（13,665 字节），为 TypeScript 源码的编译输出

### 四、总结

v0.1.3 是一个**极小的维护版本**，仅在 content-security 插件上做了简化：

| 类别 | 变化 | 影响 |
|------|------|------|
| 📝 文案简化 | 拦截消息从技术性描述改为用户友好语句 | 更好的用户体验 |
| 🧹 代码精简 | 移除标签解析、未用函数和导入 | 减少代码体积 |
| 📦 编译产物 | 新增 `dist/index.js` | 可能用于直接加载而非运行时编译 |
| ❌ 无变化 | 主进程、渲染进程、wechat-access、skills、配置模板 | 功能完全一致 |

**主进程和渲染进程与原始 v0.1.2 完全相同，无任何逻辑变化。**
